# AEON CDP+AI 项目蓝图

> 最后更新: 2026-06-02

---

## 一、项目概述

本项目包含两个层面：

### 1A. 业务项目：AEON CDP+AI 产品落地
- **客户**: AEON Digital Management Center
- **目标**: 在现有 CDP 系统上叠加 AI 能力，实现智能问数、营销助手、营销策略、AI基础后台四大模块
- **技术栈**: Copilot Studio（优先）+ NL2SQL/AI 能力集成 + Dataverse（配置数据）+ Power Platform
- **关键约束**: 业务数据在客户 CDP 数据库，非 Dataverse

### 1B. 工具项目：项目评估看板模板
- **目标**: 将本次评估过程中开发的看板工具打磨为通用模板，未来新项目可直接复用
- **定位**: 给决策者/PM 在项目评估阶段使用的可视化分析工具

---

## 二、业务项目技术栈

| 层面 | 选型 |
|------|------|
| AI Agent 平台 | Microsoft Copilot Studio |
| AI 能力集成 | NL2SQL / 文案生成 / 图片生成（取决于客户已有服务） |
| 配置数据存储 | Dataverse（知识库/Prompt 模板/策略规则） |
| 业务数据 | 客户 CDP 数据库（通过 API 集成） |
| 管理后台 | Power Platform（Power Apps） |
| 目标系统 | CDP 系统（需对接 API） |

## 三、业务功能模块（29 项）

### 3.1 智能问数（P0，7 项）
V1.0: 会员概览数据智能解读
V2.0: CDP 全报表问数覆盖 / AI 人群画像洞察
V2.1: 数据复盘 Agent / 未识别意图覆盖
远期: 多活动全量复盘 / AI 个人画像描述

### 3.2 营销助手（P1，5 项）
V1.0: AI 生成营销文案
V2.0: 自动圈选人群（标签/行为）/ 自动创建营销画布
V2.1: 语义理解圈人 / AI 营销图片生成

### 3.3 营销策略（P1，5 项）
V1.0: 通用营销方案策划
V2.0: 人群特征营销内容建议
V2.1: 营销专家库
远期: 操作说明 / CDP 指标问询

### 3.4 AI 基础后台（基础设施，12 项）
V1.0: 知识库管理 / 提示词管理 / AI 效率统计 / 策略中心（数据+画像）/ CDP AI 助手嵌入
V2.0: 核心数据解读策略升级 / 独立 AI 助手 Tab / 智能六步法提示词
V2.1: 营销复盘策略 / 问数策略 / 交互框架升级

## 五、实施方案（二选一）

### 方案一（现已改名为「阶段一」）：三大核心功能 + 基础能力底座（推荐）
- 第一阶段聚焦 3 个核心功能：
  1. **智能问数（含会员概览）** — 自然语言查询会员数据
     - **前提条件**: NL2SQL 能力由客户侧提供现成组件或第三方服务，微软仅负责集成调用，不承担 NL2SQL 引擎的开发与调优工作
  2. **AI 人群画像洞察** — 语义识别人群特征
  3. **自动圈选人群** — 对话式标签/行为圈人
- 基础底座同步建设：知识库管理 / 提示词管理 / 解读策略 / CDP AI 助手嵌入
- 双方协同：我方做 Copilot Studio + AI 集成，客户方提供 API + 数据字典
- 第二阶段扩展营销文案/策略，第三阶段做深度自动化

### 方案二（现已改名为「阶段二」）：全量并行
- 四模块 V1.0 同时启动
- 交付压力大，但客户一次看到全貌

## 六、关键风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| NL2SQL 准确率 | 高 | NL2SQL 组件由客户提供，微软仅做集成验证 |

## 七、评估看板工具（通用模板）

### 工具定位
面向 PM / 决策者的项目评估阶段工具。新项目来了之后替换数据即可复用。

### 技术栈（v3 重构）
- 前端框架: Vite + React
- 图表: ECharts（甘特图 + 影响x工作量矩阵）
- 依赖图谱: D3.js / react-force-graph（可缩放画布）
- 流程图编辑: React Flow（拖拽式节点编辑）
- 状态管理: React Context + localStorage 持久化
- 无后端，纯前端

### 工具功能清单（eval-v3 已实现）

| 功能 | 说明 | 状态 |
|------|------|------|
| Dashboard 仪表盘 | 项目健康卡片 / 待决策项 / 风险 / 依赖状态 | ✅ |
| 功能清单 | 表头排序 / 列筛选 / 行内展开 / 技术 Tag | ✅ |
| 影响x工作量矩阵 | ECharts 散点图，按模块着色 | ✅ |
| 依赖关系图 | React Flow 可缩放拖拽节点 | ✅ |
| 成本视角 | 人天 + 月运营成本，按模块汇总 | ✅ |
| 方案设计 | 方案对比表 + 可拖拽甘特图 + 功能模块编辑 + 风险编辑 | ✅ |
| 汇报模式 | 全屏大字体，投屏演示 | ✅ |
| 导出/导入 JSON | 导航栏按钮，数据可跨设备迁移 | ✅ |
| 富文本编辑器 | 加粗/斜体/下划线/文字颜色/背景色 | ✅ |
| GitHub Pages 部署 | 自动 CI/CD，在线访问 | ✅ |
| 技术路线图 | React Flow 拖拽式节点编辑器 |
| 决策日志 | 每个功能记录选型理由 |
| 汇报模式 | 全屏大字体，投屏演示用 |

### 数据字段
每个功能项包含：name / module / priority / version / risk / impact(1-10) / effort(人天) / deadline(上线时间) / urgency(自动计算) / status / desc / techAnalysis / techs(标签+百分比) / dependencies / monthlyCost / costNotes / decisionLog / diagram / note
| 方案设计 | 方案对比表 + 可编辑甘特图 + 验收标准 |
| 决策日志 | 每个功能记录选型理由 |
| 汇报模式 | 全屏大字体，投屏演示用 |

### 模板化要点
- 数据层（data.js）与视图层（app.js + style.css）分离
- 新项目只需修改 data.js 中的 DATA 和 PLANS
- 技术方案分配、风险、依赖等结构化字段通用于任何项目
- 无外部依赖（除 Mermaid CDN），纯前端
